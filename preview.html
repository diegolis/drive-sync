Drive Sync Desktop

Una app tipo InSync, pero pensada para Linux y apoyada en rclone como motor. La app no reinventa la sync: resuelve UX, configuración, estado, logs, scheduler y guardrails.

Qué resuelve

  • Agregar una o más carpetas locales
  • Elegir destino en Google Drive
  • Hacer dry-run antes de tocar nada
  • Correr sync manual o automática
  • Ver estado, errores, historial y logs

Decisión técnica

  • Tauri + React + TypeScript
  • SQLite para estado local
  • rclone como backend de sync
  • Watcher local + scheduler + lock

Modos de sincronización

copy seguro

Sube y actualiza, pero no borra en destino. Ideal para arrancar.

sync con cuidado

Espeja origen → destino. Puede borrar archivos en Drive si ya no existen localmente.

bisync difícil

Bidireccional. Acá aparecen conflictos, renombres, borrados cruzados y estados raros. No conviene soltarlo sin base sólida.

MVP v1

  • Detectar si rclone está instalado
  • Listar remotes configurados
  • Crear jobs de sync
  • Hacer preview con --dry-run
  • Ejecutar sync manual
  • Guardar historial y logs

No entra en v1

  • Resolución sofisticada de conflictos
  • Explorador completo de Drive
  • OAuth propio sin rclone
  • Tiempo real ultra fino
  • Multiproveedor cloud

Pantallas mínimas

  • Dashboard: lista de syncs, último estado, ejecutar ahora
  • Nueva sincronización: carpeta local, destino Drive, modo, intervalo
  • Preview / Dry run: qué subiría, borraría o actualizaría
  • Logs / Historial: runs anteriores y errores
  • Settings: path de rclone, logs, notificaciones

Arquitectura resumida

UI (React) ↓ Sync Orchestrator ↓ Job Scheduler + Watcher + Lock ↓ rclone (copy / sync / bisync) ↓ SQLite + Logs

Guardrails

  • No correr sync sin preview previo
  • Validar rutas absolutas
  • Impedir source = destino
  • Bloquear doble corrida del mismo job
  • Warning fuerte si puede borrar

Mi recomendación honesta

Si querés algo tipo InSync de verdad, hay que tratarlo como un producto chico, no como un script maquillado. Se puede hacer. Pero para que no nazca rota, conviene construir la base primero y soltar el bidireccional con control.
